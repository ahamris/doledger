# Devops_Ledger

Control plane for DevOps work across multiple hosts. Each host syncs its ledger to a hostname-based subdirectory to avoid merge conflicts.

## Repository layout

- **`STATUS.md`** — Generated by GitHub Action; concatenated view of all hosts' ledgers.
- **`<hostname>/`** — One folder per host (e.g. `workspace-01/`, `cl-pve-00/`), each containing:
  - `baseline.md` — Specs and constraints for that host.
  - `ledger.md` — LIFO activity log (newest on top).
  - `templates/` — Standardized config/task templates.
  - `log/` — Detailed task output and backups.

## One-time setup per host

1. Clone this repo to `/doledger-repo` (or use an existing clone):
   ```bash
   git clone git@github.com:ahamris/doledger.git /doledger-repo
   cd /doledger-repo
   ```
2. Run the setup script to create this host's directory and symlink `/doledger`:
   ```bash
   ./scripts/setup-host.sh
   ```
3. Install the `do-log` script (if not already installed):
   ```bash
   sudo cp scripts/do-log /usr/local/bin/do-log
   sudo chmod +x /usr/local/bin/do-log
   ```
4. Add your Cursor rule so the AI uses the ledger (see project docs).

## Daily use

- **Cursor / AI:** Read `/doledger/baseline.md` before tasks; use `/doledger/templates/`; after each change run:
  ```bash
  do-log "Task Description" "Success/Fail Details"
  ```
- **Ledger:** New entries are prepended to `/doledger/ledger.md`. The script commits and pushes only this host's subdirectory.

## Optional

- Point Cursor at `.cursor/rules` in your workspace for the DevOps ledger mandate rule.
